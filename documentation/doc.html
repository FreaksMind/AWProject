Abstract

The rapid expansion of open-source software development necessitates structured knowledge representation for better understanding and management of software ecosystems. WadoApp presents an ontology-driven web application that leverages GitHub’s GraphQL API to extract, structure, and analyze public software repositories. The system utilizes OWL (Web Ontology Language) to model key aspects of web application development, including programming paradigms, languages, architectures, and diverse software tools — such as frameworks. By reusing and extending existing ontologies, the proposed model provides a semantic framework that captures relationships between software artifacts, facilitating better querying, reasoning, and knowledge discovery. The application is built using Flask and rdflib for semantic data processing, with Vue.js powering an intuitive user interface for browsing and interacting with the accumulated knowledge. This system not only enables developers and researchers to explore software development trends but also supports intelligent querying and inference, offering a foundation for future applications in software analysis and recommendation systems.

Introduction

The rapid evolution of web application development has led to the emergence of diverse programming paradigms, frameworks, methodologies, and toolchains, making it increasingly difficult for developers to navigate and understand the ever-expanding software ecosystem. Platforms like GitHub host millions of public repositories, showcasing real-world examples of how different technologies and practices converge to build modern applications. However, while GitHub and similar platforms provide valuable metadata and repository statistics, they lack a semantic layer that captures the intricate relationships between software components, development techniques, and technology stacks. Without such structured knowledge, developers and researchers often struggle to identify patterns, understand architectural decisions, or explore the broader context of a given technology.

This project addresses that gap by designing an OWL-based ontology that models fundamental aspects of web development, including programming paradigms, languages, architectures, development methodologies, and an extensive taxonomy of software tools (e.g., runtime platforms, compilers, frameworks, and IDEs). By leveraging GitHub’s GraphQL API, the system dynamically retrieves and categorizes repositories, enriching the ontology with real-world software artifacts. The ontology integrates with existing knowledge models where applicable, ensuring interoperability and extensibility.

To support knowledge management and interactive exploration, we develop a Flask-based backend utilizing rdflib for semantic data processing, coupled with a Vue.js frontend for visualization and querying. The system enables users to browse structured repository metadata, perform complex SPARQL queries, and uncover relationships between technologies, providing a valuable tool for researchers, software engineers, and technology enthusiasts. Additionally, by referencing curated "Awesome Lists" and Real Life Architectures, the ontology is grounded in practical, widely recognized software ecosystems.

Ultimately, this project aims to create a powerful tool for researchers, developers, and technology enthusiasts, facilitating knowledge discovery, accelerating learning, and fostering better decision-making in software development. By uniting semantic web technologies, open-source repositories, and modern web development frameworks, the system demonstrates how ontologies can transform raw repository data into actionable, meaningful insights — paving the way for more intelligent, interconnected, and semantically aware software ecosystems.

Architecture of the Application

The architecture of the application is composed of both frontend and backend systems that seamlessly integrate to provide an intuitive and powerful tool for knowledge discovery. The frontend utilizes Vue.js, a progressive JavaScript framework, to offer an interactive and responsive user interface. The backend, built with Flask, a lightweight Python web framework, handles the data processing, ontology management, and user authentication. To handle semantic data, the application relies on rdflib, a Python library for working with RDF data, and integrates with GitHub's GraphQL API to dynamically retrieve and categorize repositories. Additionally, the application supports advanced querying via SPARQL to interact with both local ontologies and external data sources like DBpedia.

Frontend

The frontend of the application is built using Vue.js and Cytoscape.js. Vue.js is a progressive JavaScript framework designed for building user interfaces and single-page applications. Backend

Web API

The backend of the application is powered by Flask, a lightweight and flexible Python web framework. Flask handles the core application logic, processes incoming HTTP requests, and routes them to appropriate services. The backend includes multiple essential services such as a REST API, and an ontology service. A REST API (Representational State Transfer) is used to enable communication between the frontend and backend. It follows a stateless communication model, where each request from the client contains all the necessary data to be processed. REST APIs are commonly used because they are simple to implement, scalable, and widely understood. GraphQL, by contrast, offers more flexibility than REST, allowing the client to specify exactly which data it needs, reducing over-fetching.

Ontology Service

The ontology service is a critical component of the application that enables the dynamic generation, population, and querying of an ontology that models software ecosystems. The process begins with the generation of a standard OWL (Web Ontology Language) skeleton, which is an abstract structure that defines the core concepts and relationships within the ontology. A custom script using the rdflib library is responsible for creating this skeleton. rdflib is a Python library for working with RDF data, which is the foundation of the ontology. Once the skeleton is created, the service populates the ontology by retrieving relevant data from both GitHub’s GraphQL API and DBpedia. The GitHub GraphQL API provides detailed metadata about software repositories, including information about programming languages, frameworks, and contributors. Similarly, DBpedia, which is a structured dataset derived from Wikipedia, offers rich data about various concepts, including software tools and technologies. The ontology service then links this data and stores it as RDF triples, which can be queried using SPARQL. Additionally, the application includes a search bar that allows users to directly query the GitHub API. By typing in keywords or technologies, users can view top repositories that match their search criteria. These repositories are then visualized in the context of the ontology, allowing users to explore relevant software ecosystems.

Ontology of the Application

Taxonomy of the Ontology

The taxonomy of the ontology is structured to reflect the hierarchical relationships between various programming technologies and concepts. At the highest level, the ontology is divided into broad categories such as ProgrammingLanguage, Framework, OperatingSystem, and ProgrammingParadigm, each representing a distinct domain of knowledge. Within these categories, the ontology further refines concepts by defining subclasses. For instance, under the Framework class, there are subclasses like BackendFramework, FrontendFramework, and MachineLearningFramework, which capture more specific types of frameworks that are relevant to particular fields or domains.

Similarly, the ProgrammingLanguage class can include various programming languages, each characterized by their own properties such as syntax, usage, and associated frameworks. The OperatingSystem class includes operating systems that are often associated with particular frameworks or development environments, while the ProgrammingParadigm class categorizes languages and frameworks based on the paradigm they support, such as Object-Oriented, Functional, or Imperative programming.

This hierarchical taxonomy not only organizes the entities but also enables efficient querying and reasoning over the data. For example, by knowing that a specific framework is a subclass of BackendFramework, one can infer its relevance in backend development, and by understanding the relationship between a programming language and its paradigms, users can better categorize and find technologies that fit their programming style or project needs.

Structure of the Ontology

The ontology described here is designed to represent a set of concepts and relationships in the domain of programming frameworks, languages, and technologies. The ontology is built using the Web Ontology Language (OWL) and incorporates classes, properties, and relationships that define entities such as programming languages, frameworks, operating systems, and more. The structure of the ontology is generated using the Python script ontology_generator.py, which leverages the rdflib library to create the RDF graph and serialize it into OWL format. The structure of the ontology is composed of several key components.

First, the ontology defines multiple classes such as ProgrammingLanguage, Framework, OperatingSystem, License, and ProgrammingParadigm. These classes represent core entities in the system and are populated with relevant data properties such as hasName, hasDeveloper, and hasDescription, which capture basic attributes of these classes. For example, a ProgrammingLanguage class has properties like hasName and hasDeveloper, whereas a Framework class has properties like hasVersion and hasURL.

The ontology also introduces object properties, which define relationships between different classes. For instance, a Framework can be related to a ProgrammingLanguage using the usesLanguage object property. Similarly, relationships between frameworks and programming paradigms are captured through properties like supportsParadigm and usesArchitecture.

The structure of this ontology is further enriched by subclass relationships. Frameworks are divided into specific subclasses, such as BackendFramework, FrontendFramework, and MachineLearningFramework. This subclassing ensures that the ontology accommodates the various specific types of frameworks and their characteristics. For example, BackendFramework and FrontendFramework are both subclasses of the more general Framework class.

To populate the ontology with real-world data, two sources of information are used: GitHub and DBpedia. The OntologyPopulator class fetches data from these sources and updates the ontology. The GitHub data is fetched using the GitHub GraphQL API, which allows for querying repositories based on topics such as programming languages or frameworks. The fetch_repos_by_topic method allows users to specify a topic and retrieve popular repositories (e.g., repositories related to specific programming languages or frameworks). This data is then inserted into the ontology, populating entities like ProgrammingLanguage and Framework with relevant information, such as names, descriptions, and URLs.

In addition to GitHub data, DBpedia provides data on various programming languages, paradigms, and operating systems. The fetch_dbpedia_data method queries DBpedia using the SPARQL endpoint and retrieves relevant data, including labels and descriptions for programming languages, operating systems, and programming paradigms. This information is then added to the ontology using the add_dbpedia_information method. Each entity is labeled with its description and type, ensuring that the ontology is populated with authoritative data from DBpedia.

The combination of these two data sources ensures that the ontology is both comprehensive and up-to-date, reflecting real-world technologies and practices in the domain of programming. The result is a rich semantic model that can be used for knowledge representation, data integration, and querying related to programming frameworks, languages, and their associated components.

In summary, this ontology serves as a foundational model for representing the relationships and attributes of programming technologies, offering a structured view of concepts like frameworks, languages, and paradigms. The population of this ontology is powered by data from GitHub and DBpedia, providing real-world examples and references to these technologies, which enhances the utility and relevance of the ontology.
